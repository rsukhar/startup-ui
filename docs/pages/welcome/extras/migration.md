# Миграция на мажорную версию

Мажорный релиз содержит несколько ломающих изменений. Ниже — что поменялось и как обновиться.

## 0. Требования к окружению

Сначала проверьте версии зависимостей (peer dependencies) — при необходимости обновите их:

- **Vue** ≥ 3.5
- **@vueuse/core** ≥ 14
- **@inertiajs/vue3** 2.x или 3.x (нужен для `SForm`, `SUpload` и Inertia-ссылок)

## 1. Локаль по умолчанию — английская

Раньше строки были русскими. Теперь дефолт — `en`. Заметнее всего на `SDatePicker` (английские названия месяцев и дней недели) и на встроенных подписях `SConfirm`, `SPagination`, `SUpload`, `STable`/`SSelect`.

**Что сделать:** для русского интерфейса задайте локаль при регистрации:

```js
app.use(StartupUI, { locale: 'ru' });
// или без плагина:
import { configureStartupUi } from 'startup-ui';
configureStartupUi({ locale: 'ru' });
```

Подробнее — [«Локализация»](/pages/welcome/basics/localization).

## 2. Библиотека не везёт дефолты `--s-*`

CSS библиотеки больше не содержит значений переменных `--s-*` (только их использование). Приложение без своей палитры останется без цветов.

**Что сделать** — один из вариантов:

```js
// А) подключить готовую палитру
import 'startup-ui/defaults.css';
```

```scss
/* Б) задать свою палитру в проекте */
:root { --s-primary: #2563eb; /* ... */ }
```

## 3. SButton: снят `max-width: fit-content`

Кнопка по-прежнему сжимается по контенту (`display:inline-block`), но теперь ширину можно задать через `style`/класс. Если где-то полагались на жёсткий клэмп — проверьте вёрстку; для растягивания используйте проп `fullwidth`.

## 4. SForm: `titlesWidth` не влияет на титулы сверху

Инлайн-ширина заголовка применяется только при `titles-at-left`. При титулах сверху ширина больше не навязывается (это убирает лишние переносы). Обходные `:titles-width="null"` можно убрать.

## 5. SCheckboxGroup корректно эмитит `update:modelValue`

Группа чекбоксов теперь отдаёт новый массив через `update:modelValue` (раньше мутировала по ссылке). Если использовали обходные deep-watch или ручную кнопку Save из-за «неловящегося» изменения — это больше не нужно.

## 6. SFilterGroup `bind-to-query`: Inertia больше не используется автоматически

`SFilterGroup` отвязан от InertiaJS — в «чистом» виде он не требует роутера и синхронизирует URL через History API. Раньше `bind-to-query` всегда делал Inertia-визит (`router.get`) для серверного рефетча; теперь это поведение **опционально** и включается регистрацией роутера.

**Что сделать:** если используете `bind-to-query` и нужен серверный рефетч (Inertia-визит), зарегистрируйте роутер один раз при подключении плагина:

```js
import StartupUI from 'startup-ui';
import { router } from '@inertiajs/vue3';

app.use(StartupUI, { router });
```

Без регистрации `bind-to-query` продолжит работать, но только синхронизирует GET-параметры в URL (без обращения к серверу).

## Изменения API (обратносовместимые)

Эти возможности добавлены без ломающих изменений, но полезны при обновлении:

- **SSelect / SRadioGroup / SCheckboxGroup**: `options` принимает массив объектов + пропы `option-label` / `option-value` (раньше — только карта и массив пар).
- **SFormRow**: привязка поля по `name` работает на любой глубине вложенности (поле можно обернуть в `<div>`).
- **SInput**: проп `empty-value` (`'' | null`) для контроля пустого значения (напр. `''` вместо `null` для NOT NULL полей).
- **SHtmlEditor**: пропы `init`, `plugins`, `toolbar`, `menubar`, `content-style`, `content-css`, `header-offset`, событие `@init`.
- **STree**: слот `#node-actions` для действий над узлом по наведению.

## Заметка по InertiaJS

Клиент `@inertiajs/core` 3.x читает начальную страницу только из `<script data-page="app" type="application/json">`. Включите `use_script_element_for_initial_page` в `config/inertia.php`, иначе — пустой экран без ошибок.
